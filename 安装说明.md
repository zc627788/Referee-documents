# 安装和使用说明

## 📦 依赖安装

### Phase1（规则提取 + 增强规则）
**不需要额外依赖**，可以直接运行：
```bash
python src/run_pipeline.py phase1 --input tocheck_20260301.csv --output-dir data/output
```

### Phase2（GLM-4批量提取）
需要安装openai包：
```bash
pip install openai
```

或者如果使用requirements.txt：
```bash
pip install -r requirements.txt
```

---

## 🎯 当前优化成果（基于1000条测试）

### Phase1优化
- **原始不确定片段**: 1,513条
- **增强规则提取**: 125条 (8.3%)
- **过滤无姓名**: 6条 (0.4%)
- **最终AI队列**: 1,382条
- **节省**: 131条 (8.7%)

### Phase2优化
- **批量处理**: 一次API调用最多500条（不是1条1条）
- **JSON格式**: 输入输出都是结构化JSON数组
- **速度**: 1,382条 ÷ 500 ≈ 3次请求（快很多！）

## 🚀 使用流程

### 1. 运行Phase1（已优化，节省8.7%）
```bash
python src/run_pipeline.py phase1 --input tocheck_20260301.csv --output-dir data/output
```

输出：
- `data/output/xxx_result.csv` - 提取结果
- `data/output/xxx_ai_queue.csv` - AI队列（已优化，比原来少8.7%）

### 2. 运行Phase2（需要安装openai）
```bash

pip install openai

# 运行Phase2
python src/run_pipeline.py phase2 --output-dir data/output
```

## 📊 为什么AI队列还是很多？

**当前状态**（1000条测试）：
- 确定提取: 805条 (80.4%)
- AI队列: 1,382条

**原因分析**：
1. 基础规则只能处理80%的简单情况
2. 增强规则只能再优化8.7%
3. 剩余~20%的文书格式复杂，必须AI处理

**继续优化方向**：

### 方案1: 增强规则覆盖率（目标：15-20%）
当前只能优化8.7%，可以继续分析数据模式：
```bash
# 运行完整分析
python test_optimization.py
```

可能的增强模式：
- 多个连续角色（`审判长张三、审判员李四`）
- 括号标注（`审判员王五（代）`）
- 特殊格式（`审判员：张三 李四`）

### 方案2: 先运行Phase2，分析GLM-4的结果
运行Phase2后，看看GLM-4提取到了什么：
- 如果很多都提取成功 → 说明确实需要AI
- 如果很多返回null → 可以加入"无姓名"过滤规则

### 方案3: 调整基础规则的判断标准
修改`src/core/rule_extractor.py`，放宽边界字符判断
