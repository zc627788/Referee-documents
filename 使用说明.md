# 裁判文书姓名提取系统 - 使用说明

## 📋 系统概述

本系统用于从裁判文书中自动提取人名，采用**规则提取 + AI增强**的两阶段方案。

### 工作流程

```
原始数据 → 阶段1（规则提取） → 阶段2（AI提取） → 最终结果
```

- **阶段1**：规则提取（成功率85.8%，质量99.98%）
- **阶段2**：AI模型处理剩余待处理数据

## 🚀 快速开始

### 1. 执行阶段1（规则提取）

```bash
python run_phase1.py
```

这会自动执行阶段1.1的铁血版规则提取，包含：
- 预清洗（去除括号、脱敏字符）
- 多模式向量化提取
- 职称模糊匹配（代书记员 → 代理书记员）
- AC自动机双向扫描
- 铁血版清洗（STOP_CHARS熔断 + 非人名过滤）

### 2. 重置数据（可选）

如需重新提取，先重置：

```bash
python reset_ultimate.py
```

### 3. 修复多名字问题（可选）

如果发现多个名字粘连的情况：

```bash
python fix_multi_names.py
```

## 🎯 阶段1技术栈

### 核心优化

1. **预清洗**
   - 去除括号内容
   - 去除脱敏字符（X、？）

2. **多模式向量化提取**
   - 模式A: "某某某担任..." (最高优先级)
   - 模式B: "由...担任"
   - 模式C: 姓名中间有空格 "刘 红"
   - 模式D: 标准 "角色 姓名"

3. **职称模糊匹配**
   ```python
   '代理书记员': r'代(?:理)?书记员'  # 匹配"代书记员"或"代理书记员"
   '助理审判员': r'助(?:理)?审判员'
   '人民陪审员': r'(?:人民)?陪审员'
   ```

4. **positive_truncate_v2 铁血版清洗**
   - **STOP_CHARS熔断**：遇到"法/条/律/附/注"等字立即截断
   - **RISKY_SURNAMES过滤**：年/月/日/时/本/宣等高危姓氏需额外验证
   - **invalid_chars过滤**：拒绝包含"的地得与和及处理执行宣布开庭"等虚词/动词
   - **长度控制**：单姓最多3字，复姓最多4字
   - **姓氏验证**：必须以有效姓氏开头

5. **AC自动机双向扫描**
   - 基于COMMON_SURNAMES的快速扫描
   - 应用铁血版清洗

6. **全量数据二次清洗**
   - 修复历史脏数据

## 🔧 常见问题

### Q1: 为什么成功率不是100%？

剩余14.2%待处理主要原因：
1. 套话文本（无名字）："审判人员、书记员署名..."
2. 脱敏严重："书记员 ＸＸ龙"
3. 水印干扰："徐 一 马 克 数 据 网"
4. 少见姓氏："瞿德雄"（瞿不在姓氏库中）
5. 格式特殊（需AI处理）

### Q2: 如何提高成功率？

1. **扩充姓氏库**：在`src/core/rule_extractor.py`的`COMMON_SURNAMES`中添加少见姓氏
2. **使用阶段2**：剩余待处理交由AI模型处理
3. **添加水印检测**：识别并清除"马克数据网"等水印

### Q3: 发现提取错误怎么办？

1. 检查是否是脏数据（非人名）
2. 查看原文片段，分析错误原因
3. 根据错误类型调整过滤规则
4. 重新运行`python run_phase1.py`
