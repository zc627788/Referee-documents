# 快速测试指南

## 🚀 快速开始（推荐先小批量测试）

### 1️⃣ 小批量测试（100条数据）

```bash
# 创建测试输出目录
mkdir data/test

# Phase1: 规则提取（限制100条）
python src/run_pipeline.py phase1 --input tocheck_20260301.csv --limit 100 --output-dir data/test

# 查看Phase1结果
python src/run_pipeline.py status --output-dir data/test
```

**预期输出**：
- 生成 `data/test/tocheck_20260301_result.csv`（规则提取的结果）
- 生成 `data/test/tocheck_20260301_ai_queue.csv`（待AI处理的队列）
- 显示统计信息（确定提取、不确定片段等）

### 2️⃣ 测试AI提取（使用GLM-4）

```bash
# Phase2: AI精修（自动使用增强规则+GLM-4）
python src/run_pipeline.py phase2 --output-dir data/test
```

**预期输出**：
```
总计: XX 条待处理片段
[步骤1/2] 使用增强规则预过滤...
  - 增强规则提取: XX 个
  - 过滤掉无姓名: XX 个
  - 仍需AI处理: XX 个
[步骤2/2] 调用GLM-4处理剩余片段...
  GLM-4提取: [进度条]
  GLM-4成功提取: XX 个姓名
最终提取总数: XX 个有效人名
AI调用节省: XX 次 (X.X%)
```

### 3️⃣ 检查结果

```bash
# 查看最终状态
python src/run_pipeline.py status --output-dir data/test
```

打开 `data/test/tocheck_20260301_result.csv` 检查：
- 审判长、审判员、书记员等列是否正确填充
- "来源"列显示"规则"或"规则+ai"
- "flag"列显示处理状态

## 🔧 常见问题

### Q1: GLM-4调用失败怎么办？
检查 `config/config.json` 中的：
- `api_key` 是否正确
- `base_url` 是否可访问
- 账户余额是否充足

### Q2: 想加快速度怎么办？
编辑 `config/config.json`，添加：
```json
"processing": {
  "use_batch_api": true
}
```
这会启用批量API模式（一次处理多条）

### Q3: 如何只测试增强规则不调用AI？
暂时修改代码，在 `run_pipeline.py` 的 `cmd_phase2` 函数中：
```python
ai_map = _ai_batch_extract(snippets, config, use_enhanced_rules=True)
```
改为只使用规则：
```python
from core.enhanced_rule_extractor import EnhancedRuleExtractor
rule_extracted, _ = EnhancedRuleExtractor.batch_filter(snippets)
ai_map = rule_extracted
```

### Q4: 想查看具体哪些片段被提取/过滤了？
修改 `test_optimization.py`，增加输出详情的代码段。
