# 裁判文书落款提取 - 优化方案

## 方案对比

### 原方案 vs 新方案

| 维度 | 原方案 | 新方案（优化后） |
|------|--------|-----------------|
| 处理流程 | 一次性处理 | 分阶段处理，输出中间结果 |
| AI模型 | GLM-4 | **GLM-4-Flash（便宜80%）** |

## 最终方案：三阶段处理

### 阶段1：定位与规则提取
**脚本：** `step1_locate_and_extract.py`

**输入：** `原始CSV文件`

**处理流程：**
1. 读取CSV的"全文"字段
2. 尝试定位落款区域
3. 对有落款的进行规则提取
4. 计算置信度分数

**输出3个文件：**
- `stage1_high_confidence.csv` - 规则提取成功（置信度>0.8）
- `stage1_low_confidence.csv` - 规则提取不确定（置信度0.5-0.8）
- `stage1_no_signature.csv` - 未找到落款

**统计报告：**
```
总计: 10000条
├─ 高置信度: 7500条 (75%)
├─ 低置信度: 1500条 (15%)
└─ 无落款: 1000条 (10%)
```

---

### 阶段2：AI增强（低置信度文书）
**脚本：** `step2_ai_enhance.py`

**输入：** `stage1_low_confidence.csv`

**处理流程：**
1. 读取低置信度文书的落款区域
2. 调用GLM-4-Flash API提取
3. 使用JSON Schema确保输出格式
4. 合并结果

**API配置：**
```json
{
  "base_url": "https://api.vectorengine.ai/v1",
  "api_key": "your_key",
  "model": "glm-4-flash"
}
```

**输出：**
- `stage2_ai_extracted.csv` - AI提取结果
- `stage2_failed.csv` - AI也失败的（极少）

**成本估算（10万条，15%需AI）：**
- 1.5万条 × 500字 × 0.18元/1M tokens ≈ **135元**

---

### 阶段3：无落款智能处理
**脚本：** `step3_handle_no_signature.py`

**输入：** `stage1_no_signature.csv`

**智能搜索策略：**

#### 3.1 关键词定位
搜索全文中是否出现：
- "审判长"、"审判员"、"代理审判员"
- "陪审员"、"人民陪审员"
- "书记员"、"执行员"

#### 3.2 上下文提取
```python
# 伪代码
for keyword in KEYWORDS:
    positions = find_all_positions(full_text, keyword)
    if len(positions) > 0:
        # 提取每个位置前后200字
        contexts = [full_text[pos-200:pos+200] for pos in positions]
        # 合并去重
        candidate_text = merge_contexts(contexts)  # 约300-500字
```

#### 3.3 AI验证
- 如果候选区域<500字：调用AI提取
- 如果候选区域>1000字或找到>5个关键词：标记为"复杂文书，跳过"
- 如果未找到任何关键词：标记为"确实无落款"

**处理流程：**
```
无落款文书 (10000条)
  ↓
智能搜索关键词
  ├─ 找到关键词 (3000条, 30%)
  │   ↓
  │   提取上下文 → AI验证 → 提取成功
  │
  ├─ 未找到关键词 (6000条, 60%)
  │   → 标记"确实无落款"
  │
  └─ 候选区域过大 (1000条, 10%)
      → 标记"复杂文书，建议人工"
```

**输出：**
- `stage3_extracted.csv` - 从无落款中提取出的
- `stage3_truly_no_signature.csv` - 确实无落款
- `stage3_complex.csv` - 复杂文书（候选区域过大）

**成本估算（10万条，10%无落款，30%可处理）：**
- 3000条 × 400字 × 0.18元/1M tokens ≈ **22元**

## 质量检查

### 随机抽样验证
**脚本：** `step5_quality_check.py`

**功能：**
1. 从最终结果中随机抽样N条
2. 打印原文+提取结果
3. 人工核对并记录准确性
4. 生成质量报告

**使用：**
```python
python step5_quality_check.py --sample 100 --input final_result.xlsx
```
